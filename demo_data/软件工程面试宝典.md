## 一、数据结构与算法

### 1.1 常用数据结构

| 数据结构 | 特点 | 典型应用 |
|---------|------|---------|
| 数组 | 连续内存，O(1) 随机访问 | 存储固定大小的数据集合 |
| 链表 | 不连续存储，O(1) 插入删除 | LRU 缓存、多项式运算 |
| 栈 | 后进先出 (LIFO) | 括号匹配、表达式求值、DFS |
| 队列 | 先进先出 (FIFO) | BFS、消息队列、任务调度 |
| 哈希表 | O(1) 平均查找 | 缓存、去重、计数 |
| 二叉树 | 层级结构 | 搜索、排序、表达式树 |
| 堆 | 优先队列 | Top K 问题、Dijkstra 算法 |

### 1.2 排序算法复杂度

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 |
|------|---------|---------|------|-------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 快速排序 | O(nlogn) | O(n²) | O(logn) | 不稳定 |
| 归并排序 | O(nlogn) | O(nlogn) | O(n) | 稳定 |
| 堆排序 | O(nlogn) | O(nlogn) | O(1) | 不稳定 |

## 二、计算机网络

### 2.1 OSI 七层模型

从下到上：
1. **物理层**：传输比特流（网线、光纤）
2. **数据链路层**：帧传输、MAC 地址、交换机
3. **网络层**：IP 地址、路由选择、路由器
4. **传输层**：TCP/UDP、端口号、流量控制
5. **会话层**：建立/管理/终止会话
6. **表示层**：数据格式转换、加密解密
7. **应用层**：HTTP、FTP、SMTP、DNS

### 2.2 TCP 三次握手

```
客户端                    服务器
  |-------SYN------------->|
  |<------SYN+ACK---------|
  |-------ACK------------->|
  |                         |
```

1. 客户端发送 SYN，进入 SYN_SENT 状态
2. 服务器回复 SYN+ACK，进入 SYN_RCVD 状态
3. 客户端发送 ACK，双方进入 ESTABLISHED 状态

### 2.3 HTTP 状态码

- **2xx** 成功：200 OK / 201 Created / 204 No Content
- **3xx** 重定向：301 永久 / 302 临时 / 304 未修改
- **4xx** 客户端错误：400 请求错误 / 401 未授权 / 403 禁止 / 404 未找到 / 429 请求过多
- **5xx** 服务端错误：500 内部错误 / 502 网关错误 / 503 服务不可用

## 三、数据库

### 3.1 SQL 基础

```sql
-- 查询
SELECT name, age FROM users WHERE age > 18 ORDER BY age DESC LIMIT 10;

-- 连接
SELECT u.name, o.total
FROM users u
INNER JOIN orders o ON u.id = o.user_id;

-- 聚合
SELECT department, AVG(salary) as avg_sal
FROM employees
GROUP BY department
HAVING AVG(salary) > 5000;
```

### 3.2 索引优化

- **B+树索引**：适合范围查询和排序（MySQL InnoDB 默认）
- **Hash 索引**：只适合等值查询
- **最左前缀原则**：联合索引 `(a, b, c)` 只在查询条件包含 a 时生效
- **覆盖索引**：查询的列都在索引中，无需回表
- **EXPLAIN** 分析查询计划，关注 type、key、rows 字段

### 3.3 事务 ACID

- **原子性 (Atomicity)**：事务要么全做，要么全不做
- **一致性 (Consistency)**：事务前后数据满足所有约束
- **隔离性 (Isolation)**：并发事务互不干扰
- **持久性 (Durability)**：已提交事务永久保存

## 四、操作系统

### 4.1 进程 vs 线程

| 维度 | 进程 | 线程 |
|------|------|------|
| 资源 | 独立内存空间 | 共享进程内存 |
| 切换开销 | 大 | 小 |
| 通信 | IPC（管道、消息队列、共享内存） | 直接读写共享变量 |
| 崩溃影响 | 不影响其他进程 | 可能导致整个进程崩溃 |

### 4.2 死锁四个条件

1. **互斥**：资源不能共享
2. **持有并等待**：持有资源的同时等待其他资源
3. **不可抢占**：资源不能被强制剥夺
4. **循环等待**：形成资源等待环

破坏任一条件即可预防死锁。

## 五、设计模式

### 5.1 常用设计模式

- **单例模式**：全局唯一实例（数据库连接池、配置管理）
- **工厂模式**：解耦对象创建（各种工厂和抽象工厂）
- **观察者模式**：一对多的事件通知（发布订阅、事件监听）
- **策略模式**：运行时切换算法（不同支付方式、排序策略）
- **装饰器模式**：动态添加功能（Java IO 流、Python @decorator）

### 5.2 SOLID 原则

- **S** 单一职责：一个类只负责一件事
- **O** 开闭原则：对扩展开放，对修改关闭
- **L** 里氏替换：子类可以替换父类
- **I** 接口隔离：小而专的接口
- **D** 依赖倒置：依赖抽象而非具体实现

## 六、面试行为问题

### 常见问题及回答思路

1. **"介绍一下你做过的最有挑战的项目"**
   - STAR 法则：情境(Situation) → 任务(Task) → 行动(Action) → 结果(Result)

2. **"你的职业规划是什么？"**
   - 短期（1-2年）：深耕技术，成为团队可靠成员
   - 中期（3-5年）：成为某个领域的专家
   - 长期（5年+）：技术管理或架构师方向

3. **"为什么选择我们公司？"**
   - 提前研究公司业务和技术栈
   - 结合自己的兴趣和成长方向
   - 展现对公司产品的认同感
